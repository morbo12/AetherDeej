# Arduino Code Generator — Reference Structure

Each reference file follows the same structured layout to make patterns easy to scan and verify:

1. **Purpose** — What the pattern achieves.
2. **When to Use** — Scenarios where the pattern fits best.
3. **Implementation** — Code templates and usage (often labeled “Basic Template” and “Usage”).
4. **Verification** — Steps to validate behavior.
5. **Common Pitfalls & Tips** — Frequent mistakes and best practices.

Keep pattern names consistent with `generate_snippet.py` and link to examples in `examples/`.
